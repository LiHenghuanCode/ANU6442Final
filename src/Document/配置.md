回退版本很麻烦，建议万不得已别用
指令：
如果你已经把代码 push 到 GitLab 上了，现在想回退版本
# 1. 回退到上一个提交
git reset --hard HEAD~1

# 2. 强制推送到远程（注意：这会覆盖远程历史）
git push origin HEAD --force

或者
# 1. 回滚到指定提交（比如回滚上一次提交）
git revert HEAD

# 2. 或者回滚到具体的 commit id
git revert <commit-hash>

# 3. 推送到远程
git push origin main

两个都不行时，建议修改gitlab里库的git修改逻辑，使得库main分支权限可以被修改
